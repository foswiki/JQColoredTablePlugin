---++ Description

This plugin allows to colorize an html table based on certain matching rules. Coloring the table
is performed on the client side, that is javascript is used to apply the styling. This means
colors apply dynamically, but also that these won't be able to be printed out or exported in a pdf document.

The following rules can be used:

   * match keywords using string comparison
   * match keywords based on regular expression matching
   * find table cells whose content is above or below certain values
   * find table cells holding dates matching before or after a date

When a match is found css styles or classes can be applied.

---++ Examples

You type:

<verbatim>
%COLOREDTABLE{
   keyword_1="error"
   class_1="foswikiAlert"
}%
| *header* | *header* | *header* | *header* |
| data | data | data | error |
| data | data | data | data |
| data | data | data | error |
| data | data | data | data |
%ENDCOLOREDTABLE%
</verbatim>

---++ Syntax

All tables to be colorized have to be encosed in =%COLOREDTABLE= and =%ENDCOLOREDTABLE=. Configured rules apply
to all tables within. Note that =%COLOREDTABLE= sections cannot be nested. Rules are configured using

=%COLOREDTABLE{&lt;params>}=:

| *Parameter* | *Description* | *Default* |
| select | jQuery selector to find the table within the colored table section | current element |
| highlight | defines the element to colorize, possible values are =this=, =parent=, =prev=, =next=, or a jQuery selector | =this= |
| type | kind of matching operation, possible values are =string=, =regex=, =integer=, =float=, =gt=, =lt=, =date=, =before=, =after= | =string= |
| casesensitive | boolean to specify case sensitivity, =true= or =false= | =false= |
| style | default styles applied to any match | |
| class | default class applied to any match | |
| keyword_1, keyword_2, ... , keyword_n | keywords/regexes/integers/floats/dates to match | |
| style_1, style_2, ... , style_n | styles to be applied to the corresponding match | |
| class_1, class_2, ..., class_n | css class to be applied to the corresponding match | |

Any starting =%COLOREDTABLE= has to be closed by =%ENDCOLOREDTABLE= to make it work.
